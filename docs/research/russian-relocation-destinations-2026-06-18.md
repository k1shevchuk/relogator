# Направления переезда граждан РФ: рабочая база для Relogator

Дата проверки: 2026-06-18.

Документ нужен для наполнения базы стран и маршрутов. Это не юридическая консультация и не готовый публичный рейтинг. Перед показом пользователю каждая страна и каждый маршрут должны иметь отдельные источники по въезду, пребыванию, ВНЖ, документам, срокам и применимости к гражданам РФ.

## Как читать данные

Нет одного официального источника, который честно отвечает на вопрос "куда едут россияне" по всем странам сразу.

Разные источники считают разные вещи:

- первые разрешения на проживание;
- въезд или иммиграцию по гражданству;
- миграционный остаток;
- действующие ВНЖ;
- страну рождения, а не гражданство;
- только страны ЕС или ОЭСР.

Поэтому для продукта нужно хранить не один рейтинг, а карточку источника:

- `source_url`;
- `source_title`;
- `source_type`;
- `metric_type`;
- `year`;
- `citizenship_scope`;
- `last_checked_at`;
- `confidence`;
- `notes`.

## Основные источники

1. Eurostat, `migr_resfirst`: первые разрешения на проживание по гражданству, причине и стране выдачи.
   URL: https://ec.europa.eu/eurostat/databrowser/view/migr_resfirst/default/table
   Что дает: сопоставимые данные по Европе. Для 2024 года по гражданству `RU` видны страны, где граждане РФ получили первые разрешения.

2. Geostat, Migration.
   URL: https://www.geostat.ge/en/modules/categories/322/migration
   Что дает: иммигранты и эмигранты Грузии по гражданству. В таблице по гражданству есть строка `Russian Federation`.

3. Statistical Office of the Republic of Serbia, migration.
   URL: https://www.stat.gov.rs/en-us/oblasti/stanovnistvo/migracije-stanovnistva/
   Что дает: официальные данные по иммиграции и эмиграции Сербии, включая граждан РФ.

4. EMN Armenia, Country Factsheet 2024.
   URL: https://home-affairs.ec.europa.eu/document/download/b9da3d06-5999-416f-9625-d27d95153398_en?filename=2024_EMN-AM_countryfactsheet.pdf
   Что дает: первые разрешения на проживание в Армении по странам гражданства.

5. OECD International Migration Outlook / country notes.
   URL: https://www.oecd.org/en/publications/international-migration-outlook-2024_50b0353e-en.html
   Что дает: контекст по странам ОЭСР и отдельные страновые заметки. Для Турции полезно как контекст по изменению числа россиян с ВНЖ.

6. UN DESA International Migrant Stock 2024.
   URL: https://www.un.org/development/desa/pd/content/international-migrant-stock
   Что дает: долгосрочный запас мигрантов. Это не поток переезда и не ВНЖ, поэтому использовать только как справочный контекст.

## Что показывать пользователю

Не показывать страну как "простую" или "сложную" вообще. Сложность считается для связки:

`пользователь + цель + страна + маршрут`

Одна и та же страна может быть:

- простой для короткого въезда;
- средней для проживания 3-12 месяцев;
- сложной для ВНЖ;
- очень сложной для работы, бизнеса, семьи или гражданства.

## Требования к данным страны

Минимальная карточка страны:

```ts
type CountryResearchStatus =
  | "reviewed"
  | "needs_review"
  | "stale"
  | "reference_only"

type CountryResearchEntry = {
  countryCode: string
  countryName: string
  priority: 1 | 2 | 3
  researchStatus: CountryResearchStatus
  sourceUrls: string[]
  sourceMetricNotes: string[]
  appliesToCitizenship: "RU"
  lastCheckedAt: string
}
```

Минимальная карточка маршрута:

```ts
type RouteAvailability = "available" | "conditional" | "blocked" | "unknown"

type RouteDimension =
  | "documents"
  | "cost"
  | "approvalRisk"
  | "speed"
  | "adaptation"

type RouteEvaluation = {
  routeId: string
  availability: RouteAvailability
  dimensions: Record<RouteDimension, 1 | 2 | 3 | 4 | 5>
  blockers: string[]
  unlocks: string[]
  sourceIds: string[]
  lastReviewedAt: string
}
```

## Следующие действия для разработчика

1. Расширить `CountryCode` и базу стран.
2. Добавить статусы глубины данных: `reviewed`, `needs_review`, `stale`, `reference_only`.
3. Не показывать неподтвержденные страны как проверенные.
4. Перестать фильтровать все неподходящие маршруты через `return null`.
5. Ввести статусы маршрута: доступно сейчас, условно доступно, нужно подготовить, не подходит.
6. Добавить блок "Что может открыть больше вариантов" на результатах.
7. Добавить шкалы документов, расходов, срока подготовки, риска отказа/дополнительного запроса и адаптации.
8. Для каждой карточки страны и маршрута показывать дату проверки и источники.
